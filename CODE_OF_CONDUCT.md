# Code of Conduct

## Our Standard

Contributors are expected to be respectful, direct, and technically honest.
Critique the code and assumptions, not the person.

## Expected Behavior

- Provide evidence-based technical feedback.
- Assume good intent, but be precise about risks and regressions.
- Keep review discussion focused on correctness, safety, and maintainability.
- Respect privacy and do not repost private data from issues or artifacts.

## Unacceptable Behavior

- harassment, intimidation, or personal attacks
- publishing private credentials, tokens, chat IDs, or personal data
- hostile, bad-faith review behavior
- spamming or low-effort promotional issues/PRs

## Enforcement

Maintainers may remove, edit, reject, or block contributions that violate these
rules.
